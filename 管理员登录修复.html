管理员登录修复

用于处理“Invalid login credentials / 密码错误 / 不是管理员”等问题。

先确认网站文件

1

必须上传修复版网站包

仓库根目录应直接看到 index.html、app.js、styles.css、config.js 和 supabase 文件夹。

2

不要上传外层 site 文件夹

打开网站包后，把里面的文件上传到仓库根目录。上传后建议强制刷新浏览器，或用无痕窗口测试。

修复 Supabase 管理员

1

检查用户

进入 Supabase → Authentication → Users，查找 adminpzh@math.example。

  • 没有该用户：选择 Add user → Create new user，不要选择 Send invitation
  • 有该用户：确认邮箱已确认，并在用户菜单中重置密码
2

设置登录信息

Email: adminpzh@math.example
Password: pass2008@@

创建时启用 Auto Confirm User。网站输入的登录名仍为 adminpzh。

3

运行权限修复 SQL

进入 SQL Editor，运行 supabase/管理员检查与修复.sql。最后一条查询应显示 role 为 admin。

完成后，关闭旧网页标签，重新打开网站并登录。首次登录成功后立即在“账号设置”修改初始密码。

不同错误代表什么

Invalid login credentials：账号不存在、密码不一致，或账号不是可用的密码登录用户。Supabase 为安全起见不会进一步区分。

Email not confirmed：邮箱尚未确认。

账号验证成功，但尚未取得管理员权限：Auth 登录正常，但 profiles 表中的 role 不是 admin；运行修复 SQL。

无法连接 Supabase：检查 config.js 的 Project URL 和 Publishable key。

不要把 service_role 或 secret key 放进 config.js、GitHub 仓库或网页源码。
