推荐：覆盖原 GitHub 仓库

1

先备份原仓库

可以下载仓库 ZIP，或保留当前分支，以便回退。

2

解压修复版网站包

解压 数之谜_GitHubPages网站_修复版.zip，确认第一层直接是 index.html、app.js、styles.css、config.js。

3

删除旧网站文件后上传新文件

把修复版包内全部文件上传到仓库根目录并覆盖。不要上传外层文件夹，也不要把首次导入题库 ZIP 上传到公开仓库。

4

等待 Pages 更新后强制刷新

打开网站，用无痕窗口测试，或在桌面浏览器按 Ctrl+F5。手机端可清除该网站缓存后再打开。

修复管理员登录

1

运行主数据库脚本

Supabase → SQL Editor，完整运行 supabase/schema.sql。

2

创建或重置管理员用户

Supabase → Authentication → Users。必须存在邮箱 adminpzh@math.example 的用户。创建时选择 Create new user，设置密码并启用 Auto Confirm User；不要使用 Send invitation。

网站登录名: adminpzh

3

运行管理员权限修复

在 SQL Editor 运行 supabase/管理员检查与修复.sql。若第一条查询没有用户，先回到 Authentication 创建用户，再重跑。

4

重新登录

关闭旧标签页，重新打开网站。登录成功后进入账号设置修改密码。

Supabase 的“Invalid login credentials”不会明确区分用户不存在与密码错误，因此必须在 Authentication → Users 中直接检查。

导入题库

1

进入批量导入

管理员登录 → 管理后台 → 批量导入。

2

选择首次导入包

选择完整交付包 setup 目录内的 首次导入题库_42题.zip。导入完成前不要关闭页面。

重复导入会按题号覆盖更新，不会重复创建同一题。

检查是否部署正确

  • 手机：顶部图标应为正常小尺寸，底部显示题目/搜索/登录导航。
  • 平板：题目卡片通常为两列，详细页自动变为单列。
  • 电脑：宽屏题目卡片最多三列，管理表格可完整操作。
  • 登录失败：页面会显示中文原因，并可打开“管理员登录修复说明”。
